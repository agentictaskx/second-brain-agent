### Ruby Feedback — Channel Conventions
- **Every post = a DSAT** from a user reporting a product issue
- **Issue categories:** ranking, aigc-gem, aigc-image, uiux, feature, quality, timeliness, safety
- **Comments = follow-up:** discussion, root cause analysis, next steps, ICM/ADO links, owner assignment
- **When summarizing:** Use a single flat table (1 header row + N data rows) with all DSATs. Do NOT split into separate sections per thread. Include: date, reporter, DSAT summary, category, follow-up from comments, ICM, ADO.
- **Tracking:** Look for ICM links, ADO links, and owner assignments in comments. Flag gaps where no tracking exists.
